## Non-obvious design decisions

### 1. Always resolve offsets to numeric via kadm before consuming
`kgo.NewOffset().AtStart()` / `AtEnd().Relative(-N)` failed on the target cluster — the implicit `ListOffsets` resolution inside `kgo` consistently hit `context deadline exceeded` (partition `-1`, i.e. client-level), even though the brokers were healthy.

The fix in `internal/kafka/consumer.go` is to always resolve offsets ourselves via `kadm.ListStartOffsets` / `ListEndOffsets` / `ListOffsetsAfterMilli`, then pass explicit `kgo.NewOffset().At(N)`. This bypasses the bug entirely.

The `SeekEnd` mode also no longer means "HWM - 1"; it now means "show the last `maxMessages / numPartitions` records per partition" so users see history immediately instead of waiting for new traffic.

### 2. Host-alias dial-time rewrites
Many internal Kafka clusters advertise hostnames like `broker-2:9093` that don't resolve from the user's machine. Instead of forcing each user to edit `/etc/hosts`, every `kgo.NewClient` is built with a custom `kgo.Dialer` that intercepts `host:port` and substitutes the IP from `Profile.HostAliases` (`broker-2 → 192.0.2.20`).

Stored per-profile, persisted in `profiles.json`, applied to both the manager's long-lived client and the transient consume-client (`consumer.go`).

### 3. Stale connection on profile edit
`SaveProfile` auto-disconnects the profile, so the next Connect uses fresh `kgo.Client` with updated bootstrap / aliases. Without this, editing aliases on a connected profile silently reused the old dialer and broker-discovery errors reappeared.

### 4. Auto-refresh on Topics tab only
TopicsPage runs a single 10 s master interval that does:
1. `ListTopics`
2. `MessageRates` for all current topic names
3. For each expanded topic: `GetTopicPartitions` + `ListGroupsForTopic`
4. Calls `props.onTick?.()` which the parent uses to refresh cluster info

Refresh is *silent* — fetched data replaces the existing data, the `kind: "loading"` state never shows (so no UI flicker). Loud "loading" state only appears on initial expand / explicit user retry.

Interval is created once per `profileId` change; `topics`, `expanded`, `onTick` are mirrored to refs so they don't re-create the timer. The cleanup `clearInterval` fires when TopicsPage unmounts (= user navigates to another tab), so polling stops automatically.

### 5. ListTopics vs ListTopicsWithInternal
`kadm.ListTopics` excludes internal topics (`__consumer_offsets`, `__transaction_state`, …) by default. We use `ListTopicsWithInternal` in `admin.go::ListTopics` so the "내부" column actually has rows to mark as `✓`.

### 6. Topic selection lifted to App.tsx
`Consume` and `Produce` pages share the same `sharedTopic` state at the App level. Picking a topic in one tab carries to the other. Resets to `""` whenever the selected profile changes (topic names are not cross-profile valid).

### 7. Resizable UI primitives
`frontend/src/components/ResizableColumns.tsx` exports two reusable pieces:

- `useColumnWidths(storageKey, columns: ColumnDef[])` + `<ResizableTh>` — for tables. A column with `grow: true` (currently only the Consume grid's `Value`) gets no resize handle and auto-fills remaining space; the table is `width: 100%` + `tableLayout: fixed`, so grid responds to window resize naturally.
- `useResizableWidth(storageKey, default)` + `<VerticalSplitter direction="ltr"|"rtl">` — for panel splitters. Used by:
  - App sidebar (ltr): drag right grows sidebar
  - Consume preview panel (rtl): drag right shrinks preview

All sizes persist in `localStorage` and double-clicking any handle resets to default.

### 8. Visible column borders + invisible-but-large drag hit area
`.col-resizer` is 8 px wide and transparent (so it doesn't compete with the gray `border-right`). On hover/active, the resizer paints a 3 px centered accent-color line. The always-visible boundary comes from the `<th>` / `<td>` `border-right` (scoped to `table.resizable-grid` so other tables aren't affected).

## File-by-file responsibilities

### Backend
- `main.go` — Wails entrypoint, embeds `frontend/dist`, sets window options (`Title: "Kafka Client"`, 1280x820), wires `OnShutdown` to `app.shutdown` (CloseAll)
- `app.go` — `App` struct holding `*profile.Store` and `*kafka.Manager`; every exported method = IPC endpoint
- `internal/profile/profile.go` — `Profile` struct + JSON store at `~/.kafka-client/profiles.json`. `HostAliases map[string]string` is part of the Profile.
- `internal/kafka/manager.go` — `Manager` (pool keyed by profileID), `Client` (servers, aliases, kgo client, kadm client), `aliasDialer`
- `internal/kafka/admin.go` — `TopicSummary`, `BrokerInfo`, `PartitionDetail`, `ClusterInfo`; `ListTopics` (uses `ListTopicsWithInternal`), `ListBrokers`, `GetTopicPartitions`, `GetClusterInfo`
- `internal/kafka/topic_admin.go` — `TopicConfigEntry`, `TopicDescribe`; `DescribeTopic`, `CreateTopic`, `DeleteTopic`, `UpdateTopicPartitions`, `AlterTopicConfigs`
- `internal/kafka/consumer.go` — `SeekMode`, `ConsumeOptions`, `Message`; `Consume` (explicit numeric offset resolution, drain detection, hex preview for binary, base64 for raw bytes)
- `internal/kafka/producer.go` — `ProduceRequest`, `ProduceResult`; `Produce` (single-message via `ProduceSync`)
- `internal/kafka/groups.go` — `GroupView`, `GroupMemberView`, `GroupPartitionLag`; `ListGroupsForTopic` (calls `kadm.ListGroups` + `kadm.Lag(all)` then filters to groups committing to the requested topic)
- `internal/kafka/rates.go` — `MessageRate`; `MessageRates` (current end offsets - offsets-at-(now-60s), divided by 60)

### Frontend
- `App.tsx` — Sidebar with profiles list + ContextMenu, topbar with cluster + controller, tab buttons (Topics/Consume/Produce/Settings), shared `sharedTopic` state between Consume and Produce
- `pages/TopicsPage.tsx` — auto-refresh master loop, tree rows (RowGroup → ExpandedTopic → PartitionsCard + GroupsCard), context menu, create/edit/delete dialogs
- `pages/ConsumePage.tsx` — resizable columns grid, regex/key/value/headers filter, virtual scroll, JSON export, resizable preview panel
- `pages/ProducePage.tsx` — single-message form + dismissible help banner
- `pages/SettingsPage.tsx` — language toggle, config folder, import/export
- `components/ProfileDialog.tsx` — name / bootstrap servers / host aliases textarea / Schema Registry URL / default topic
- `components/TopicCreateDialog.tsx`, `TopicEditDialog.tsx`, `TopicDeleteDialog.tsx`
- `components/ContextMenu.tsx` — generic right-click menu (used by sidebar profiles + topic list)
- `components/ResizableColumns.tsx` — `ColumnDef`, `useColumnWidths`, `<ResizableTh>`, `useResizableWidth`, `<VerticalSplitter>`
- `lib/i18n.ts` — ko / en translation dictionaries + `t(lang, key, vars?)`
- `lib/errors.ts` — `errString(unknown)`
